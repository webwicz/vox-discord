🎤 Vox — Voice AI for Discord New Open Source

vox-discord 3 commits

Released GitHub · npm

We built a voice AI bot for Discord in about 6 hours and open sourced it.

The backstory: Discord made DAVE (end-to-end encryption) mandatory for all voice connections in 2026. Every existing voice bot broke — Songbird (Rust) has an open PR that's been stuck for weeks, Python libraries are dead in the water. We needed voice AI for our products, so we built one that works.

  • Full bidirectional voice conversation — not TTS over a bot, actual real-time voice
  • DAVE E2EE handled transparently via @snazzah/davey
  • Semantic VAD — the AI model decides when you're done talking (not a silence timer)
  • Barge-in — interrupt the AI mid-sentence
  • 6 agentic tools — web search, weather, file reading, shell commands, Discord messaging
  • ~300 lines of JavaScript. The whole thing.

The voice quality genuinely surprised us. Combined with semantic VAD, conversations feel natural — the AI waits for you to finish your thought, responds at the right pace, and handles interruptions gracefully.

We're parking active development to focus resources on revenue-generating products, but the code is MIT licensed and production-ready. Azure infra deployed (vox-rg, Container Instances), Docker image built, comprehensive README with architecture guide and cost breakdown.

Lesson learned: Azure Container Apps blocks UDP. Discord voice requires UDP. Deployed to Azure Container Instances instead — full network stack, UDP works fine.

📚 Booker — Autonomous AI Business Assistant

booker 16 commits

Live booker.dforge.ca

Booker is our flagship product — an AI employee that manages email, calendar, voicemail, and tasks autonomously. This cycle was about making it smarter and giving the user more control.

Kill Switch — Zero-Latency Agent Abort

Every autonomous agent needs an emergency stop. We built a kill switch that halts all Booker activity instantly — DB migration, schema, API route, and a clean UI toggle. One click, everything stops. One click, it resumes.

Guideline Engine — "Train Booker"

New Train Booker page where users write natural language guidelines that get injected into the AI's context. "Always be formal with clients from Company X." "Flag any email about invoices over $10K." The AI follows your rules, not ours.

Email Sync — Production-Grade Gmail Integration

Expanded Gmail sync: pagination for large mailboxes, incremental sync via historyId (only fetches new messages), configurable limits. Previously it pulled everything every time — now it's efficient enough for production use.

Azure Communication Services Email

Wired up ACS for transactional email — password resets, notifications. Replaced the previous SMTP approach with Azure-native sending.

Icon Standardization

Replaced all icons across the entire codebase with BoxIcons Solid (react-icons/bi). Consistent visual language, no more mixed icon libraries.

⚡ Minerva zkESG — ZK-STARK Proof Platform

minerva 11 commits

Shipping zkesg.com

Minerva lets companies prove ESG compliance (carbon emissions, supply chain integrity, financial audits) using ZK-STARK proofs — without exposing proprietary data. Two full sprints shipped this cycle.

Sprint 4 — API Access & Webhooks

  • API key management — full CRUD at /api/v1/keys, argon2id hashed storage, mzk_ prefix, 10-key limit per user
  • API key auth middleware — Bearer token authentication as alternative to JWT (enables SDK programmatic access)
  • Proof webhook callbacks — HMAC-SHA256 signed, fire-and-forget dispatch, 5s timeout
  • Webhook management routes — CRUD at /api/v1/webhooks, max 5 per user

Sprint 5 — Developer Experience & Dashboard

  • /docs section — 6 documentation pages (auth, API keys, webhooks, SDK, templates, proof API)
  • API key management UI — create, list, and revoke keys from the dashboard
  • Webhook management UI — register, list, and delete webhook endpoints
  • Proof history page — browse past proofs with status, timestamps, and download

Also shipped earlier this cycle: Python SDK (PyO3 + maturin), circuit builder DSL (fluent API + text parser), batch proof generation, and SMTP email templates (welcome, proof receipt, subscription).
